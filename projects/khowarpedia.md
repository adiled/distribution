# khowarpedia

**Tier:** T5 (Cultural / dormant) • **Repo:** https://github.com/adiled/khowarpedia • **License:** *not declared* • **Stars:** 0 • **Live site:** https://adils.me/khowar/

## What it is

khowarpedia is a static, browser-side Khowar to English / Urdu dictionary lookup served from `adils.me/khowar/`, with around 8,200 entries from Rehmat Aziz Chitrali's Khowar Academy corpus (the same data behind webonary.org/rachitrali), plus a small crowdsourced extension and a small captdjtobrien extension, all merged into a single Fuse.js index in the browser. It runs with no backend. The page is one HTML file, one CSS file, three JSON datasets, an Awami Nastaliq font from SIL, jQuery, and Fuse 6.4.6 from a CDN. The dictionary table renders right to left in Awami Nastaliq at 26px, with per-alphabet filter chips down the left rail and a fuzzy-search box up top that toggles between two lookup modes (`KHW`, which keys on `lexeme`, `latin`, and Arabic-script `word`; and `EN/UR`, which keys on `translations.english` and `translations.urdu`). An "Explorer" checkbox loosens the Fuse threshold from `0.01` to `0.3` for browse-style discovery instead of exact lookup. Mode and threshold persist in `localStorage`, and the dataset is cached client-side and only re-fetched when `LAST_UPDATED` advances.

The footer reads `Made with ❤️ from The Village White Rock, Booni ⛰️`. Booni is a town in Upper Chitral. That line and the attribution beside it ("Chitrali, Rehmat Aziz - Khowar Academy") are the project's only first-person voice. There is no README. The GitHub `description` field carries the marketing line: ":open_book: Search friendly Khowar - English dictionary using fuzzy search. Neat UX: forkable and adaptable to any language dataset with semantics."

The data is organized by Khowar alphabet, not by Arabic alphabet. Per-letter JSON files in `dataset_rachitrali/` use the lexicographer's transliterations (`alif`, `bay`, `pay`, `tay`, `ttay`, `chay`, `hhay`, `khay`, `daal`, `ddaal`, `zaal`, `ray`, `rray`, `zay`, `zhay`, `seen`, `sheen`, `soad`, `zoad`, `toay`, `zoay`, `aen`, `ghain`, `fay`, `qaaf`, `kaaf`, `gaaf`, `laam`, `meem`, `noon`, `vao`, `hay`, `hhhay`, `yey`, `yyey`, `dzay`, `tsay`, `chghay`, `zghay`, `thsay`, and a couple of others), 42 alphabet positions in total. The Latin transliteration is generated by `to-latin.js`, a 30-line lookup table that normalises diacritics from the source lexemes (`ɕ` and `ċ` to `ch`, `ʐ`/`ʑ`/`ż`/`ž` to `z`, `x` to `kh`, `ʂ`/`ș`/`š` to `s`, `ȡ` to `d`, `ȶ`/`ţ` to `t`, and so on) so the search index can be hit by Latin typists who only have an English keyboard.

## Why this dossier exists

khowarpedia is deliberately out of the agentic-systems thesis that drives the rest of the portfolio. It is a cultural project with a language-preservation function and a Booni-and-Chitral-rooted audience, and its distribution lives in a different orbit from `clwnd`, `ccft`, `iksir`, `orch`, `rishta-lang`, and `opencode-dir`. Pushing it through Show HN, `r/LocalLLaMA`, `awesome-claude-code`, or any other agent-systems channel would mis-route it onto an audience that has no use for a Khowar lookup, and would dilute the thesis-aligned launches with a cultural-project beat they cannot frame.

The dossier exists to record three things. The project is not forgotten; it is in T5 by design, not by neglect, and it ships maintenance pulses every twelve to twenty months. If a moment opens (a research collaboration with Stockholm or Chicago, a grant, a partnership with FLI or AKES, a Wikipedia incubator handover), the project is ready to be re-tiered into something active. And the agent-systems portfolio's distribution work should not contaminate the dictionary's audience by importing the wrong channel mix.

## State

### Code and data, current

One HTML file, one CSS file, one app.js (287 lines), one to-latin.js (30 lines), one deploy.sh (a single-line `rsync` to `adil@adils.me:/var/www/adils.me/khowar`). Three datasets, all flat JSON arrays with the same item schema (`alphabet`, `word`, `lexeme`, `grammar`, `translations[]`, `latin`).

- `dataset_rachitrali/all-latin.json` is the primary corpus, 8,213 entries, 1.4 MB, derived from Rehmat Aziz Chitrali / Khowar Academy via `webonary.org/rachitrali`. The same 8,213 entries are also broken out per Khowar letter into 42 individual JSON files. `all.json` (the Arabic-script-keyed equivalent) is also in the directory.

- `dataset_captdjtobrien/all-latin.json` is a 7-entry extension corpus added 2025-01-18 ("new dataset", commit `c0b332e`) and lightly extended on 2026-03-29 ("more", commit `fe54f80`). The shape is the same as the others.

## Audience

Three concrete kinds of people, none of which are addressable through agent-systems channels.

The first is the Khowar speaker in Chitral, Booni, Mastuj, Drosh, Garam Chashma, or the Ghizer / Yasin valleys of Gilgit-Baltistan, looking up a word they half-remember and want spelled correctly in Awami Nastaliq before sending it in a WhatsApp or Facebook reply. This person reads the right-to-left table natively, types Urdu on their phone, and prefers a Khowar-first toggle (`KHW` mode searches `lexeme`, `latin`, and Arabic-script `word`). Existing online resources for this person are the Webonary rachitrali interface (read-only, slow on mobile, account-gated for editing), Anusha's 500-word list (small, dated), and the FREELANG download (Windows / Android only, not browsable in a browser). khowarpedia is the only working browser-side Khowar lookup the maintainer is aware of that loads instantly on a 2G connection after first visit, runs offline once the dataset is cached, and renders with the SIL Awami Nastaliq font that is specifically engineered for South Asian minority-language Nastaliq.

The second is the Khowar-speaking diaspora in Karachi, Lahore, Islamabad, Peshawar, the Gulf states, and Western metros, whose children are growing up English- or Urdu-dominant and whose family chats want a usable Khowar lookup the kids will actually open. This audience reaches the project through Facebook groups, X handles tied to Chitrali identity, the Aga Khan Council communities, and word of mouth at family gatherings. Reach here is social and personal, not list-mediated.

The third is the academic and documentary linguistics audience: Henrik Liljegren's Hindukush Areal Typology project at Stockholm University, Elena Bashir's South Asian Languages and Civilizations group at Chicago, the SIL Pakistan branch (which co-holds the Webonary rachitrali copyright with Rehmat Aziz Chitrali and which built the Awami Nastaliq font that the project ships), the Forum for Language Initiatives (FLI) in Islamabad which has direct programmatic relationships with Khowar communities, and the Endangered Languages Project (ELP) which already catalogues Khowar and accepts community-resource submissions. This audience treats khowarpedia as a working artifact in the small list of online Khowar resources, alongside the Webonary, the Sloan print dictionary, the Anusha word list, and the FREELANG download. Inclusion in these registries (Glottolog provider list, ELP resources, OLAC, Lexilogos's Khowar page) is the channel that compounds quietly over years.

The audience that does not overlap cleanly: Hacker News, `r/LocalLLaMA`, `awesome-claude-code`, opencode Discord, Show HN, X / Twitter agent-tooling threads. Pushing khowarpedia through any of these would land it on readers who have no Khowar context and would frame it as "another dictionary site", which is not the value it provides to the actual audience.

## Distinguishing claim

Two claims, both defensible from the code and the artifact landscape.

The first claim is that khowarpedia is the only freely-available, browser-native, bidirectional Khowar lookup that combines the rachitrali corpus (the largest open Khowar dataset, around 8,200 entries) with a Latin-transliteration index, an Awami Nastaliq render, an alphabet-filter rail, and a one-click contribute path that lands a pull-request on the dataset directly. Webonary rachitrali has the same corpus but is server-side, slower on cellular, and not editable without an SIL Webonary account. Sloan's print dictionary is in print only. Anusha is small. FREELANG is download-only. Lexilogos points at all of the above without being one itself. Nothing in the existing online Khowar landscape combines the four properties (corpus size, browser-native lookup, transliteration round-trip, low-friction contribution).

The second claim is the data shape. The schema (`alphabet`, `word`, `lexeme`, `grammar`, `translations[].urdu`, `translations[].english`, `latin`) is structured and consistent across all 8,213 entries, which means the dataset has reuse value beyond the dictionary front-end (downstream NLP work, mother-tongue education materials at AKES schools, FLI literacy projects, Wikipedia's Khowar incubator, academic citation). The maintainer is not making this claim publicly. The dataset just sits there, structured. That is unusual for a community-edited Khowar resource.

## Distribution state, gaps

The license is not declared. This is the one launch-readiness gap that matters across both the code (MIT or similar) and the dataset (CC-BY-4.0 with attribution to Rehmat Aziz Chitrali / Khowar Academy / SIL). Per `policy.md` §4, a project shipping without a license is publishing nothing. For a language-preservation artifact specifically, the dataset license is more sensitive than the code license: the rachitrali source is published under a Webonary CC license with SIL co-attribution, and a downstream redistribution needs to honour that. The code license can be MIT and stand alone. The dataset license needs a one-paragraph note in the repo (a `DATA-LICENSE.md` or a section at the bottom of the eventual `README.md`) that names the rachitrali source's terms, the SIL co-copyright, and the attribution chain, before any non-incidental redistribution. Today no LICENSE exists for either.

There is no README. A four-paragraph README would carry: what the project is, who maintains it, what dataset it draws from and under what terms, how to contribute (already wired in the footer link), and how to run locally. The absence of a README is the single biggest usability gap for the second audience (diaspora) and the third audience (linguists). The first audience (Khowar speakers using the site) does not see the README; they see the live page.

The project is not in any registry that catalogues Khowar resources. Glottolog, the Endangered Languages Project, OLAC, Lexilogos's Khowar page, and Omniglot's Khowar page all have slots for "online dictionaries / lookups". khowarpedia could be listed in each. The submission processes are slow but the slots persist for years.

The Khowar Wikipedia incubator (`incubator.wikimedia.org/wiki/Wp/khw`) is led by Rehmat Aziz Chitrali, the same person whose corpus khowarpedia draws from. The two efforts are natural collaborators. No outreach has happened.

Social presence is zero. There is no Twitter / X handle for the project, no Facebook page, no Instagram. The Chitrali diaspora's primary social surfaces are Facebook groups (Chitral-themed community pages, university-alumni groups, AKES alumni communities) and WhatsApp circles. Reach into those is personal-network mediated and not addressable through public channels.

There is no acknowledgement of the SIL Awami Nastaliq dependency in any visible artifact. The font is shipped as `static/AwamiNastaliq-Regular.woff2` under the SIL Open Font License. The OFL needs the SIL copyright and reserved name to travel with the font, which means a small `static/OFL.txt` and a font-attribution line on the page or in a future README.

What surprised me at step 4, when writing the project's-own-voice description.

How tightly the project knows itself. With no README, no docs, no AGENTS.md, no SPEC.md, the artifact's voice still arrives intact through three signals: the footer line ("Made with ❤️ from The Village White Rock, Booni ⛰️"), the attribution to Rehmat Aziz Chitrali, and the per-letter Khowar-alphabet file naming. These three together communicate the project's claim more clearly than most README-driven projects. The maintainer is from there, the source is named, the data is organised by the language's own structure rather than by Arabic-script collation. The November-2024-to-March-2026 maintenance pulses (the visual rehaul, the explorer mode, the typo fix on the contribute link) show the project is alive at the cadence its audience can absorb, which is annual at most.

The other surprise was how cleanly the SIL thread runs through the entire project. The rachitrali corpus is co-copyrighted with SIL Global on Webonary. The Awami Nastaliq font shipped in `static/` is from SIL's Pakistan-minority-language font program. The Endangered Languages Project that catalogues Khowar is co-developed with SIL. The maintainer has not surfaced this anywhere visible, but the artifact is structurally a SIL-adjacent project whether it says so or not. Future distribution beats that involve the dataset should treat SIL as a participant in the ecosystem and reach out by courtesy ahead of any wider push, not as a gatekeeper but as a collaborator-by-default.

The third surprise was confirming that the existing online Khowar dictionary landscape is genuinely thin. Webonary rachitrali is the only other open digital Khowar lookup of meaningful corpus size, and it is server-side and account-gated for editing. Sloan's Khowar English Dictionary is print only. Anusha is small. FREELANG is download-only. Lexilogos and Omniglot are aggregator pages. khowarpedia's claim, that it is the only browser-native bidirectional Khowar lookup with a low-friction contribute path, holds in 2026 as it did in 2021 when the project shipped. That is not common for a five-year-old static site, and it is the reason the project is worth keeping alive even at T5 cadence.
